5. Lists#

Today, we’ll introduce lists, our first data structure!

But first, let’s recap:

5.1. Recap#

Variables

You can think of a variable as a box in storage that contains a thing (or things). The name of the variable is a label on the box. When you use a variable in code, you get the value out of the box.

A variable can hold almost anything in Python. But there are a few simple data types that will cover most of the data we work with:

  • Numeric - ints and floats

  • String - any textual data, denoted by quotes (’’ or “”). Any character can be put into a string.

  • Boolean - True or False

Functions

Functions are our actors; they perform actions.

Python is a stickler for grammar, so be careful to follow the syntax for a function EXACTLY! (this applies to all structures in Python)

if-elif-else

    if <boolean condition>:
        <do this>
    elif <a different boolean condition>:   # only if the above condition is False
        <do this instead>
    else:                                   # only if all of the above conditions are False
        <do this>

5.2. Lists#

(Read WTOP Ch 6)

A list is an ordered grouping of data. So if a variable is a box, a list is a box of boxes!

  • enclosed in brackets with items separated by commas [A, B, C, D]

  • can contain any object type (even another list!) and types don’t need to be consistent within the list

A = [1,2, 'hello', 3.5, False]

print(A)

[1, 2, 'hello', 3.5, False]

5.2.1. Adding to lists#

You can add A SINGLE item to the end of a list with .append(item). However, this item could be a list. ‘Append’ updates the list in place.

You can concatenate two lists with +.

A.append('celery')
print(A)

[1, 2, 'hello', 3.5, False, 'celery']

B = ['DS', 256]
print(A+B)
print(A)
print(B)

[1, 2, 'hello', 3.5, False, 'celery', 'DS', 256]
[1, 2, 'hello', 3.5, False, 'celery']
['DS', 256]

A.append(B)
print(A)

[1, 2, 'hello', 3.5, False, 'celery', ['DS', 256]]

5.2.3. Indexing of lists#

The locations in a list are indexed by number, starting at 0. That is, a list starts with a zero-th entry.

Alternatively, you can index lists from the end backwards starting with -1 (the last item in a list).

new_list = ['alpha', 'beta', 'gamma', 'delta', 'epsilon', 'zeta', 'theta']
print(len(new_list))

You extract a value from a list using brackets. For example, myList[5] would return the 6th entry of the list.

And you can use indexing to assign new values to items in the list.

print(new_list[5])
print(new_list[-2])

5.2.4. Slicing#

A slice is an extraction of a sub-list from a list.

You slice a list as:

new_list[startIdx:stopIdx:stepSize]

Notes:

  • the list will go up to but not include the stopIdx.

  • any of the parameters can be skipped.

    • No first index will start from the beginning (e.g. new_list[:5] goes from the beginning to the 4th element)

    • No second element will go to the end (e.g. new_list[2:] will start at the 2th element and go to the end)

    • No last element just selects all values sequentially

    • Skipping the first two elements will go from the beginning to the end (e.g. new_list[::3] selects every third element starting from the beginning)

STRINGS CAN BE SLICED AS IF THEY ARE LISTS OF CHARACTERS! HOW FUN!

    hi = 'hello'
    bad_place = hi[:4]

5.2.6. Finding an element in a list#

Is the element in the list?

To check if an element is in a list, use the keyword in like this:

```python
   item in new_list
```

This returns a Boolean value, T/F.

Where in the list is the element?

If a list contains an element, you can find where that element is using .index(searchItem, startIdx, endIdx).

  • index will only return the first instance it finds in the specified range.

  • If the element is not in the list, the function will return an error. Two options to avoid this error:

    • Check first with and if statement using in

    • Use try-except (see below)

Try-Except

Try-Except are a special kind of conditional statement used to catch and handle errors.

   try:
       <some code>
   except:
       <code on how to handle error>

Typically, an error stops the code from running any further. However, in a Try-Except, if the code in a try block fails then the code in the except block runs; basically, the code is saying “try this and if it doesn’t work, do this instead”. This is most often used for error handling, to catch mistakes that might have been introduced earlier in the code. The except block usually displays some message alerting the user that the code above has triggered an error.

5.3. Tuples#

Tuples are just like lists EXCEPT:

  • They are immutable, meaning once they are created you can not change them.

  • They are identified by parantheses () instead of brackets [].

When functions return multiple values, those values are stored as a tuple by default.

my_tuple = (1,2,'hello')
my_eplut = my_tuple[-1::-1]
print(my_eplut)

('hello', 2, 1)

new_list[3] = 'D'
print(new_list)

['alpha', 'beta', 'gamma', 'D', 'epsilon', 'zeta', 'theta']
